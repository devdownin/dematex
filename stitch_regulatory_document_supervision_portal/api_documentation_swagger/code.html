API Reference & Core Integration

High-authority programmatic access to the Sovereign Ledger. Integrate your existing regulatory reporting systems using our secure, RESTful architecture.

Listing (Cursor-based)

Efficiently traverse large datasets using opaque cursors. Our listing API avoids the performance pitfalls of offset pagination, ensuring stable reads across millions of records.

GET /v1/documents CURSOR_REVERSE

limit

Integer (1..100)

starting_after

String (UUID)

cURL Example content_copy
curl -X GET "https://api.ledger.gov/v1/documents?limit=10" \
  -H "Authorization: Bearer [API_KEY]"

Delta API (Incremental)

Synchronize local state with the global ledger by fetching only what changed since your last sync. Ideal for high-frequency monitoring and real-time compliance dashboards.

update

Change Streams

Stream document status updates without polling. Uses standard Server-Sent Events (SSE) for low-latency notifications.

history

Sequence Numbers

Every record includes an atomic sequence number (X-Seq). Use this to resume syncs from specific points in time.

Unitary Access (Idempotency)

Reliable write operations. Our API uses Idempotency-Key headers to prevent duplicate processing of the same request, crucial for ledger integrity. 

LedgerClient client = LedgerClient.builder()
    .apiKey("sk_test_51Mz...")
    .build();

RequestOptions options = RequestOptions.builder()
    .setIdempotencyKey("62ec219f-074a-4713-b605")
    .build();

Document doc = client.documents().create(params, options);
